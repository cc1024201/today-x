# 刷猫高手 (Brush Master) — 设计文档

**日期：** 2026-04-26  
**灵感来源：** Brush Jjaemu（2026年4月爆火的韩国刷猫反应游戏）  
**目标：** 完整复刻核心机制，加入换肤系统与连击倍率，单文件浏览器游戏

---

## 1. 核心玩法

角色在三个状态间循环，玩家需要在正确时机操作：

| 状态 | 描述 | 玩家操作 |
|------|------|---------|
| **BACK** | 角色背对玩家 | 按住鼠标/触摸，持续刷毛得分 |
| **TURNING** | 角色开始转头（警告阶段） | 准备松手 |
| **LOOKING** | 角色盯着玩家 | 必须已松手，否则死亡 |

**死亡条件：** 进入 LOOKING 状态后，未在容错窗口内松手（超时仍在刷毛）→ YOU DIED。

---

## 2. 得分与连击系统

### 得分公式

```
得分 += 刷毛时间(ms) × 倍率 / 100
```

### 连击倍率

- **计数单位：** 角色完成一次 LOOKING、玩家成功松手 = 1 次连击
- **升级节奏：** 每 3 次成功升一级

| 连击次数 | 倍率 |
|---------|------|
| 0–2 | ×1 |
| 3–5 | ×2 |
| 6–8 | ×4 |
| 9–11 | ×8 |
| 12+ | ×16（封顶，计数继续跳动给予反馈） |

- **重置：** 被抓一次倍率归 ×1

## 3. 皮肤系统

### 数据结构

```js
{
  id: string,
  name: string,
  emoji: string,
  colors: { bg, primary, accent },
  timing: {
    backMin: number,      // BACK 最短时长(ms)
    backMax: number,      // BACK 最长时长(ms)
    turningMin: number,   // TURNING 最短时长(ms)
    turningMax: number,   // TURNING 最长时长(ms)
    reactionWindow: number // LOOKING 容错窗口(ms)
  },
  fakeTurnChance: number, // 假转头概率(0–1)
}
```

### 四套皮肤参数

| 皮肤 | BACK 时长 | TURNING 时长 | 容错窗口 | 假转头概率 | 特色 |
|------|-----------|-------------|---------|-----------|------|
| 🐱 橙猫 째무 | 2000–4000ms | 600ms | 500ms | 0% | 基准手感，原版复刻 |
| 😴 睡觉领导 | 1000–5000ms | 400–800ms | 500ms | 0% | 高随机，最难预判 |
| 👁️ 监考老师 | 1500–3000ms | 400ms | 250ms | 30% | 最快容错 + 假转头 |
| 📷 监控摄像头 | 固定 2000ms | 固定 500ms | 500ms | 0% | 固定节奏，机械恐惧 |

### 假转头行为（监考老师专属）

- 进入 TURNING 时抛硬币（30% 概率），决定本次是假转头，到达中点时回到 BACK
- 玩家如已松手不计连击、不扣血
- 视觉：耳朵/眼睛动到一半停止，重新背对

## 4. 界面流程

```
MENU
  ├─ 开始游戏 → [首次游玩? → TUTORIAL] → PLAYING
  ├─ 选择皮肤 → SKIN_SELECT → MENU
  └─ 显示历史最高分

PLAYING
  └─ 死亡 → DEAD

DEAD
  ├─ 再来一局 → PLAYING（复用当前皮肤，跳过选肤）
  └─ 主菜单 → MENU
```

### 各界面内容

**主菜单：** 游戏标题、开始游戏按钮、选择皮肤按钮、历史最高分

**皮肤选择：** 4 张皮肤卡片（名称、emoji、难度标签），当前选中皮肤高亮

**首次引导（仅首次）：** 3 秒或点击跳过，说明「背对刷，转头停」规则

**游戏中 HUD：**
- 左上：当前得分
- 右上：连击倍率（`×4 连击!`），倍率升级时弹出动画

**YOU DIED：**
- 暗色背景 + 愤怒表情
- 红色 YOU DIED 大字
- 本局得分 / 历史最高分 / 本局最高连击倍率
- 「再来一局」按钮

## 5. 技术架构

### 技术栈

单文件 `index.html`，零依赖，零构建

### 渲染分层

```
z-index 高  [HTML div 层] 角色表情、UI 组件、HUD
            [Canvas 层]   背景、粒子动效、刷毛轨迹
z-index 低
```

角色朝向/旋转动画用 JS 直接操作 div `transform style`（不依赖 CSS transition），保证与游戏状态机同步。

### 输入处理

统一使用 **Pointer Events API**（`pointerdown` / `pointerup`），一套代码覆盖鼠标与触摸，避免 touch/mouse 双触发问题。

### 动画驱动

所有时序逻辑用 `deltaTime`（毫秒）累积，不用帧计数，保证 60fps 与 120fps 设备难度一致。

### 音效

Web Audio API 程序化生成，三类音效：
- 刷毛中：OscillatorNode 白噪 + 低频调制持续循环
- 转头警告：短促提示音
- 死亡：冲击音效

### 持久化

`localStorage` 存储两个键：
- `brushmaster_highscore`：历史最高分（number）
- `brushmaster_tutorial_seen`：首次引导是否已展示（boolean）

## 6. 实现优先级

1. 核心状态机 + Pointer Events 输入（先跑通橙猫单皮肤）
2. 手感调优（TURNING / LOOKING / 容错窗口，直到「感觉对了」）
3. 连击倍率 + HUD
4. YOU DIED 画面 + localStorage
5. 其余三套皮肤
6. Web Audio 音效
7. 首次引导 TUTORIAL
8. 移动端适配调试
