# AI 还是人？(Turing Test Quiz) — 设计文档

**日期：** 2026-04-26
**灵感来源：** "Your AI Slop Bores Me" 等 AI 判别类互动，契合当下满屏 AI 内容的社会情绪
**目标：** 单文件浏览器答题游戏，展示文字片段，玩家猜 AI 或人类写的，追连击最高记录

---

## 1. 核心玩法

屏幕中央一张卡片显示一段文字（100 字以内），下方两个按钮：

| 按钮 | 操作 |
|------|------|
| **AI 写的** | 判断为 AI 生成 |
| **人类写的** | 判断为人类创作 |

点击后卡片做 3D 翻转：
- 正面变**绿色** → 答对
- 正面变**红色** → 答错
- 背面显示正确答案 + 一句 `hint` 解析（如"AI 常用'馈赠''回甘'这类空洞意象"）

停留 1.5 秒后自动滑入下一张。

**连击机制：**
- 答对 +1，答错归零
- 记录历史最高连击，存入 `localStorage`
- 答错时连击数字抖动动画，强化反馈
- 游戏不中断，连击归零后继续答题

## 2. 题库

### 规格

| 字段 | 类型 | 说明 |
|------|------|------|
| `text` | string | 展示给玩家的文字，≤100 字 |
| `isAI` | boolean | `true` = AI 生成，`false` = 人类写的 |
| `hint` | string | 翻牌后显示的一句解析 |

### 数量与分类

共 **20 道**精心策划的题目（质量优先于数量），四类各约 5 道：

| 分类 | 特征 |
|------|------|
| 朋友圈 / 微博风格 | 日常感受、生活碎片 |
| 产品评论 | 购物评价、使用体验 |
| 新闻 / 时评句子 | 正式表达、观点陈述 |
| 短诗 / 散文 | 意象与情感 |

### 难度设计原则

- AI 例子：用词空洞泛化、结构对称整齐、情感平稳、缺乏具体细节
- 人类例子：有具体细节（人名、地名、数量）、口语化、情感有温度或矛盾
- 刻意选让人犹豫的例子，而非一眼即判的

题库在 JS 中以数组硬编码，随机洗牌后循环。

## 3. 界面流程

```
MENU
 └─ 点击「开始」→ PLAYING

PLAYING（无结束状态，答错继续）
 └─ 展示卡片 → 点选 → 翻牌揭晓（1.5s）→ 下一题
```

### 主菜单

- 标题：**AI 还是人？**
- 副标题：你能分清吗？
- 历史最高连击展示
- 「开始挑战」按钮

### 游戏中 HUD

- 左上：当前连击数（`🔥 连击 × N`）
- 右上：历史最高（`最高 N`）
- 中央：文字卡片
- 下方：「AI 写的」「人类写的」两个按钮

## 4. 技术架构

### 技术栈

单文件 `ai-or-human/index.html`，零依赖，零构建

### 状态机

```
STATE = { MENU, PLAYING }
```

### 核心逻辑命名空间 `TT`（Turing Test）

```js
TT.shuffle(arr)           // Fisher-Yates 洗牌
TT.isCorrect(guess, isAI) // 判断答对
TT.getNextStreak(current, correct) // 返回新连击数
```

### 渲染

- 无 Canvas，纯 HTML + CSS
- 卡片翻转：CSS 3D `rotateY`，`transform-style: preserve-3d`，`backface-visibility: hidden`
- 卡片正面：题目文字
- 卡片背面：正确答案（AI / 人类）+ hint
- 答错时连击数字：`@keyframes shake`
- 新卡片进场：`@keyframes slideInUp`

### 持久化

`localStorage` 存一个键：
- `tt_best_streak`：历史最高连击（number）
